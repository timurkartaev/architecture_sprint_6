1. Системы партнеров обращаются к core-app напряму, а не через gateway.
2. Сервис core-app нагружен запросами с разных систем, к нему идут запросы со стороны ins-comp-settlement, пользователей и партнеров. 
3. Из-за интеграции core-app партнерами и запросов от пользователей, может появиться риск долгой работы запросов, в схеме нет CQRS механизма, к нему ходят партнеры и за данными аналитики.
4. Высокая нагрузка на сервис для учета клиенстких данных (client-info) со стороны пользователей(UI) и core-app
5. База данных развернута в единственном экземпляре
6. Один из сервисов может создать больше кол-во соединений к БД из-за чего другие сервисы не смогу подключиться
7. ins-product-aggregator использует синхронное взаимодействие back-to-back
8. Сервис ins-product-aggregator получается избыточное кол-во запросов с core-app ins-comp-settlement
9. Для сервиса ins-product-aggregator не используются ни один паттерн, Retru, Circuit breaker, Rate limiter и timeout.
